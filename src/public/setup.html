OpenFang on Railway

Deploy once, finish setup in the browser.

This wrapper keeps OpenFang persistent on a Railway volume, handles auth handoff into the native dashboard, and gives you a safe overlay file for any advanced config from the OpenFang docs.

Status

Fast Path

1. Keep these variables set `SETUP_PASSWORD`, `OPENFANG_API_KEY`, plus one provider key.
2. Open the native dashboard The buttons above inject the OpenFang API token into browser storage for you.
3. Use the built-in OpenFang wizard Save provider keys, create agents, and configure channels from the real app.

Environment

Advanced Config Overlay

Edit /data/config.user.toml here. It is included automatically by the generated base config, so you can use any valid section from the OpenFang configuration reference without touching the generated bootstrap file.

Leave wrapper-owned root keys alone: include, home_dir, data_dir, api_listen, and api_key. Everything else is fair game.
